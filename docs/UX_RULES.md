# UX Rules – BEE112

## MOBILE FIRST (CRITICAL)

- Mobile is NOT desktop scaled down
- On mobile:
  - Logo/text BEE112 (left)
  - Hamburger menu (right)
  - Navigation hidden by default
- On desktop:
  - Inline navigation
  - No hamburger

## CTA CLARITY

- One primary CTA per screen
- CTA must be obvious and readable
- CTA must never compete with navigation

## VISUAL COMFORT

- Avoid large uninterrupted white areas
- Sections must be visually distinct
- Reading should be effortless on mobile

## SIMPLICITY

- No unnecessary animations
- No visual tricks
- No UX experimentation

Predictable > Fancy
